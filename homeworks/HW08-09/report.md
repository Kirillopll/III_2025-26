# HW08-09 – PyTorch MLP: регуляризация и оптимизация обучения

## 1. Кратко: что сделано

- Использован датасет **EMNIST (split="balanced")**, содержащий рукописные цифры и буквы.
- В части **A** сравнивались базовая модель MLP без регуляризации, модель с Dropout, модель с BatchNorm и вариант с EarlyStopping.
- В части **B** исследовано влияние слишком большого и слишком маленького learning rate для Adam, а также поведение SGD с momentum и weight decay.

---

Использован датасет:

## 4. Базовая модель и обучение

Использована архитектура MLP:

784 → 512 → 256 → 128 → 47

Активация:

ReLU

Функция потерь:

CrossEntropyLoss

Оптимизатор:

Adam (lr = 1e-3)

Batch size:

128

Регуляризация:

Dropout  
BatchNorm  
EarlyStopping

---

## 5. Часть A (S08): регуляризация (E1-E4)

Проведены эксперименты:

E1 — базовая модель без регуляризации

E2 — добавлен Dropout

E3 — добавлен BatchNorm

E4 — BatchNorm + EarlyStopping

Наблюдения:

Dropout уменьшает переобучение модели.

BatchNorm улучшает стабильность обучения и ускоряет сходимость.

EarlyStopping позволяет остановить обучение в момент лучшего значения validation accuracy.

Лучшая модель: E4.

---

## 6. Часть B (S09): LR, оптимизаторы, weight decay (O1-O3)

Проведены эксперименты:

O1 — слишком большой learning rate (Adam, lr = 1e-1)

O2 — слишком маленький learning rate (Adam, lr = 1e-5)

O3 — SGD + momentum + weight decay

Наблюдения:

слишком большой learning rate вызывает нестабильность обучения

слишком маленький learning rate сильно замедляет обучение

SGD с momentum показывает результат близкий к Adam

weight decay улучшает обобщающую способность модели

## 8. Анализ

Без регуляризации модель начинает переобучаться:
train loss уменьшается, а val loss перестаёт улучшаться.

Dropout уменьшает переобучение и повышает стабильность.

BatchNorm ускоряет обучение и делает кривые более плавными.

EarlyStopping позволяет остановить обучение на оптимальной эпохе.

Слишком большой learning rate приводит к нестабильности функции потерь.

Слишком маленький learning rate делает обучение слишком медленным.

SGD с momentum показывает хорошие результаты, но требует больше эпох.

Оптимальный learning rate ≈ 1e-3.

## 9. Итоговый вывод

Регуляризация улучшает качество модели и уменьшает переобучение.

BatchNorm показал наилучшее влияние на стабильность обучения.

EarlyStopping позволяет выбрать оптимальное количество эпох.

Оптимальный learning rate является важным параметром обучения.

Лучшая модель:

MLP + BatchNorm + EarlyStopping.

Модель показывает хорошую точность на датасете EMNIST.
