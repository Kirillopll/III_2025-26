# HW12 – временные ряды: temporal split, baseline-модели и GRU-прогноз

## 1. Кратко: что сделано

В работе использовался датасет временного ряда `S12-hw-dataset.csv`.  
Цель — спрогнозировать значение целевой переменной `target` на один шаг вперёд.

Данные были разделены по времени на `train / validation / test` без перемешивания.

Сравнивались модели:

- B1 — наивная модель (предыдущее значение)
- B2 — скользящее среднее
- B3 — Ridge regression с лаговыми признаками
- R1 — нейросеть GRU

Лучшей моделью оказалась GRU (R1), показавшая наименьшую ошибку по метрике MAE на validation.

## 4. Temporal split и признаки

### 4.1. Разбиение по времени

Использовано разбиение:

train = 70%  
validation = 15%  
test = 15%

Разбиение выполнялось строго по времени.

Random split некорректен для временных рядов, потому что модель может получить информацию из будущего.

- lag_14

## 5. Модели и эксперименты (B1, B2, B3, R1)

B1 (naive-last):

прогноз равен предыдущему значению ряда.

## 7. Анализ

Наивный baseline показал хороший результат, что характерно для временных рядов с зависимостью от предыдущих значений.

Moving average показал худший результат, так как усреднение сглаживает изменения и может терять важную информацию.

Использование лаговых признаков позволило модели Ridge учитывать зависимость от прошлых значений.

GRU показала лучший результат, поскольку способна учитывать последовательные зависимости во временном ряде.

При работе с временными рядами важно избегать утечки данных, поэтому используется temporal split.

Выбор модели производится по validation, а test используется только для финальной оценки.

Ошибки модели связаны с резкими изменениями временного ряда.

## 8. Итоговый вывод

Для простых временных рядов могут использоваться базовые модели.

Нейронные сети позволяют учитывать сложные зависимости во временных рядах.

Temporal split является обязательным при работе с временными рядами.
